MarkDown 文档排版规范

Posted on 2022-01-09 Edited on 2022-01-11 In 杂项

MarkDown 文档排版规范

Markdown 是由 John Gruber 于 2004 年创建的一种文本标记语言，被设计用来 专注于文本写作。

针对 MarkDown 文档的语法规范问题，开源平台 GitHub 发布了 Markdown GFM（GitHub Flavored Markdown） 标准规范，并修改了 MarkDown 解析器。 规范全文链接： GitHub Flavored Markdown Spec。

目前针对 MarkDown 文档的简体中文排版，也已经有比较详细的规范，这篇微信公众号文章已经做了详细介绍：基于 Markdown 的中文文档排版规范，公众号内容大部分来自于 GitHub 用户 Tunghsiao Liu 的规范文档，简体中文版的文档链接：中文文案排版指北。

目前中文排版规范中待完善的方面

中文文档中的英文书籍名、报刊名和文章标题（此条已有规范，但未推广普及）

根据新闻出版行业标准的编辑规范： [CY/T 154-2017](http://sxqx.alljournal.cn/uploadfile/sxqx/20190304/CY T154—2017 中文出版物夹用英文的编辑标准.pdf) 标准第 10 条，应该按以下方案处理：

  • 英文书籍名、报刊名：使用英文斜体
  • 英文文章标题：使用英文正体字，并以引号标示
  • 英文书籍的副书名：使用冒号和正书名隔开
  • 英文文章的副标题：使用英文破折号和标题隔开

双引号前是否应该加空格

应该加，否则视觉上会分不清，不管是直角引号还是弯角引号都应该加。

直角引号和弯角引号的使用争议

针对繁体中文和简体中文排版（竖体和横体），同时互联网字体排版方面的问题，争议较大。知乎网站上的讨论链接：中文为什么要用直角引号？。既然争论难以统一。个人建议应该共存，同一篇文章、同一份刊物、同一本书籍，风格统一即可：

  • 直角引号：一级引号（『』）和二级引号（「」），但港台地区是反过来的
  • 弯角引号：一级引号（“ ”）和二级引号（’ ’）

使用 VS Code 编辑 MarkDow 文档

笔者同时使用 Typora 和 VS Code 这两个工具，前者是非常优秀的 MarkDow 文档编辑工具。

如果你采用后者，强烈建议安装好插件：markdownlint，此插件作者是：David Anson，GitHub 链接。

在VS Code 中按这个插件的各项规则编辑和修改 MarkDown 文档即可。

2021 下半年软考及格分数线确定

Posted on 2022-01-05 Edited on 2022-01-06 In 软考高项

2021 年下半年软考合格标准

合格标准已于昨日公布，初级、中级、高级各科依然是 45 分（各科目满分 75 分的 60%）

以下为通知原文，网页链接：关于2021年下半年计算机技术与软件专业技术资格考试合格标准有关事项的通告 (mohrss.gov.cn)

经此通知发布，2021 年下半年的软考告一段落。已经通过的考生，等待领取查询和领取证书即可。而需要再战的考生，需要重新出发了

部分地区 45 分以下也可以拿证

目前，陕西和青海的考生，如果成绩在 45 分以下，还有可能拿到 “省证”。因为陕西、青海两省有省合格标准，至于具体的合格标准，以当年的省软考办具体通知公告为准

2021 年上半年，青海省软考合格标准为：高级总分不低于 130 分、单科成绩不低于 35 分；中级和初级总分不低于 85 分、单科成绩不低于 35 分。陕西省软考合格标准并没有包含所有资格考试，而是根据当年考试情况定出省分数线。例如：信息系统项目管理师各科目的分数分别为：基础知识 41 分、案例分析 42 分、论文 42 分

这两省的省证目前是电子证书，证书仅在本省内有效

部分地区已提供电子版证书

目前证书电子版尚未全国范围内提供，广东、浙江、山东省已实行电子证书，可在本省的人事考试网等网站下载。其中山东省从 2021 年开始提供

证书编号查询、电子文件查询、纸质证书领取时间

证书编号查询：合格标准公布后，大概 10 个工作日左右开通查询

电子文件查询：合格标准公布后，大概 20 个工作日左右后，在考试所在省的人事考试网查询

纸质证书领取：一般在考试结束 4 个月开始领取，本期预计是在 2022 年 3 月底或 4 月初陆续发放（各地区时间可能不一致，以当地软考办发送的短信通知为准）

2022 年上半年软考高项考情展望

Posted on 2021-11-24 Edited on 2021-12-24 In 软考高项

标题里的软考就是：全国计算机技术与软件专业技术资格 (水平) 考试 ^官方网站，高项是：信息系统项目管理师，属于高级资格 ^资格设置
由于项目管理的通用性，每年报考高项的考生是各项高级职称中最多的
此项考试每年在 5 月和 11 月进行，一般是在周六全天（2022 年上半年的考试预计日期是 5 月 28 日），在一天内完成三个考试科目：综合知识、案例分析、论文，三个考试科目的满分均为 75 分，及格分数一般为 45 分（需要三科同时达到及格分数）

考试方法：
三个考试科目均为笔试
综合知识为单项选择题，共 75 道题目，需要涂答题卡，由机器阅卷
案例分析为问答题，一般为 3 道答题，每道答题由 4 - 5 道小题组成，需要在答题纸上答题，由人工阅卷
论文写作为写作题，从 2 个选题中选择一个写作，也需要在答题纸上撰写，由人工阅卷

2021 年 11 月的高项软考中，有以下变化

第 1 章（信息化和信息系统）占分比例由 2021 年上半年的 17 分增加为 19 分（包含时政题）
时政题目由 2021 年上半年的 1 分，变更为 5 分，预计比例不会变化
由于时政题目归属第 1 章，所以去除时政题后，第 1 章的占分将由 2021 年上半年的 16 分变更为：14 分

2021 年 11 月考试中未变化的部分

项目管理（第 2 章至第 14 章，包括第 16 章）在综合知识的占分比例：2021 年上半年和下半年均为：35 分
第 27 章（管理科学基础知识）在综合知识的占分比例：2021 年上半年和下半年均为：5 分
案例分析符合考前预估，未有变化
论文符合考前预估，其中之一为进度管理

下期考试：2022 年 5 月份考试备考建议

  1. 注意时政题目的复习，例如：十四五规划、5G 规划、个人信息保护法等信息化相关政策文件的学习
  2. 综合知识的重点仍然在：信息化和信息系统（20 分左右）、项目管理知识（40 分左右）、管理科学基础知识（5 分左右）
  3. 保持对项目管理 10 大知识领域考点的重点复习，综合知识和案例分析考试中均为重要考点，不掌握会导致考试不通过，计算题的各项公式要熟练掌握
  4. 论文写作依然需要提前多练习，熟悉架构和撰写速度，否则很难在 120 分钟之内完成论文字数要求
